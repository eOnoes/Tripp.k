# 85% Read-Only Planning/Review Readiness Milestone Card v0.1

Status: future milestone card. This does not change the current 80% read-only Goose replacement estimate.

This milestone defines a future gate beyond the current 80% readiness level.

## Scope

Structured and moderately ambiguous read-only planning/review only.

This milestone does not include edit/build replacement, live writes, approval/apply runtime behavior, or broad Goose parity.

## Required Proofs

- Session variety pack passes across at least three distinct read-only scenario families.
- Partial-evidence synthesis remains useful without overclaiming.
- Branch rolloff preserves relevant older context while keeping current direction primary.
- Blocked outcomes remain visible longer than ordinary findings when still relevant.
- Cross-surface coherence holds across TASKS, Current Understanding, and Cyst.
- Operator-independence pack artifact passes across the full variety pack.
- Release discipline artifacts remain stable:
  - scoped beta statement
  - known limitations
  - surface-role guidance
  - Gate GO / NO GO disclaimer
  - mock/planning-only evidence disclaimer

## Blockers

- broader session variety still requires Goose interpretation
- partial-evidence summaries overclaim
- older blocked outcomes disappear too early
- older relevant branch context is erased instead of compressed
- docs or UI drift into write-readiness or broader parity wording
- operator-independence artifact passes only on a narrow subset of flows

## Invalidation Conditions

- cross-surface contradiction returns
- mock/planning-only evidence is described too strongly
- blocked-state visibility regresses in longer sessions
- branch rolloff rewrites history or loses needed context
- beta wording drifts beyond read-only planning/review scope
- future write docs begin to imply current runtime capability

## Out Of Scope At 85%

- edit/build replacement
- live writes
- approval/apply runtime behavior
- patch/apply workflows
- broad Goose parity
- open-ended engineering reasoning parity
- runtime mutation of any kind
