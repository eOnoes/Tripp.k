# Read-Only Long-Session Stress v0.1

Status: required evidence lane for the internal scoped 90% read-only planning/review claim.

## Purpose

This proof checks whether Tripp.g remains compact, coherent, and reconstructable through a longer read-only session with branch shifts, repeated blocked outcomes, partial evidence, and aging context.

## Exact Scenario Steps

Use a 10-task read-only session:

1. Planning-only retrieval suggests two plausible directions.
2. Inspect Branch A for initial useful context.
3. Run safe shell for bounded environment/repo support.
4. Trigger blocked shell or escalation for the first blocked outcome.
5. Inspect Branch B so relative usefulness changes.
6. Follow-up inspect related to Branch B.
7. Planning-only retrieval refinement introduces another possible related path and remains non-authoritative.
8. Trigger blocked shell or escalation again for the second blocked outcome.
9. Inspect older related file or docs/config source to create aging/rolloff pressure.
10. Review the Read-Only Gate as scoped harness verdict only.

## Expected Current Understanding

### What We Know

- show only current most useful branch/facts
- include one compressed older-branch line when still relevant
- do not list every task

### What Remains Uncertain

- keep mock/planning-only uncertainty visible
- mention newly suggested but uninspected paths
- acknowledge that current understanding changed over time

### Blocked In Read-Only Mode

- preserve blocked outcomes clearly
- if multiple blocked events are still relevant, summarize them compactly

### Next Read-Only Direction

- remain one clear next direction
- favor current strongest branch
- stay provisional

## Pass Criteria

Pass if:

- Current Understanding remains compact and coherent
- older useful branch context compresses but does not vanish when still needed
- blocked outcomes remain visible
- mock uncertainty remains visible
- next direction stays clear
- operator can reconstruct the session from Tripp surfaces without Goose help

## Fail Criteria

Fail if:

- summary turns into a task dump
- older relevant context is lost
- blocked outcomes disappear too early
- synthesis sounds more certain over time without evidence
- next direction becomes vague or contradictory
- operator still needs Goose to explain what changed over the session
